# Delegation Rules

## Principle

If the task would inflate the main thread with technical detail, delegate it.

## Delegate by Default

- code exploration across multiple files
- strategy research
- implementation work
- testing/backtesting
- multi-file refactors
- reviews and adversarial validation
- exchange / venue investigation

## Inline Only

- route the task
- summarize results
- consult Engram
- verify a tiny amount of config or prompt content

## Goal

Keep the main thread thin. Keep durable knowledge in Engram.
